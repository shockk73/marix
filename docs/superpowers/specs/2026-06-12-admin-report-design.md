# Дизайн: админ-отчёт по сессиям юзеров

**Дата:** 2026-06-12
**Автор:** brainstorming session
**Порядок:** под-проект 5 из 6

## Цель

Админ просит «покажи отчёт по юзерам» → бот присылает HTML-файл со всеми
юзерами и их последними 50 сообщениями с LLM.

## Решения, принятые на brainstorming

| Решение | Значение |
|---|---|
| Триггер | LLM-tool `generate_sessions_report` (только `role=admin`) |
| Формат | Один самодостаточный HTML-файл, документом в TG |
| Объём | Последние 50 `chat_messages` на юзера (= `LLM_HISTORY_SIZE`) |
| Зависимости | Нет: stdlib `html.escape` + f-string шаблон |

## Содержимое отчёта

Для каждого авторизованного юзера (сортировка: последняя активность сверху):

- Шапка: имя из `user_profiles` (или «без имени»), `user_id`, роль,
  дата авторизации, число активных слежек.
- Активные слежки одной строкой каждая: провайдер, направление, дата, окно, интервал.
- Диалог: последние 50 `chat_messages` хронологически —
  `user` / `assistant` пузырями с таймстампами (`created_at` → локальное время),
  `tool_calls` и `tool`-результаты свёрнуты в `<details>` (имя инструмента + JSON).
- Всё содержимое через `html.escape`.

Шапка отчёта: дата генерации, число юзеров, версия `LLM_SESSION_VERSION`.
Минимальный встроенный CSS (тёмная тема, читаемые пузыри, моноширинный JSON).

## Изменения в коде

1. `db.py`: `get_authorized_users() -> list[dict]` (user_id, role, authorized_at);
   переиспользуются `get_recent_chat_messages(user_id, 50)`, `get_user_watches`,
   `get_user_name`.
2. Новый модуль `report.py`: `build_sessions_report() -> str` (HTML-строка).
   Чистая функция от данных — легко тестировать.
3. `llm/tools.py`: схема + хендлер `generate_sessions_report`. Хендлер строит HTML,
   шлёт `BufferedInputFile(html.encode(), filename="sessions-YYYY-MM-DD.html")`
   через бота (бот доступен инструментам — добавить `bot` в `ToolContext`),
   возвращает агенту `{"sent": true, "users": N}`.
4. Доступ: инструмент в списке только для админа + проверка роли в хендлере.

## Тесты

- `build_sessions_report`: юзер с сообщениями/слежками → HTML содержит имя,
  пузыри, `<details>` с tool-вызовом; экранирование (`<script>` в сообщении
  юзера не исполняется — приходит экранированным).
- Юзер без сообщений → секция «нет сообщений».
- Не-админ вызывает инструмент → ошибка «только для админа».

## Вне скоупа

- Пагинация, фильтры по датам, выгрузка в другие форматы, веб-интерфейс.
- Маскировка паролей делается на этапе сохранения истории (спека автоброни),
  отчёт дополнительно ничего не маскирует.
